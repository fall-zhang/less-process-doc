## 记一次 tauri 升级

### 升级前提

在 tauri2 正式版已经发布半年多之后，才决定从 tauri 1 升级为 tauri v2，

<br />

### tauri 的变动

配置文件的变更，原来的一个包被拆分为多个包，更好的去开发 tauri 的插件，以及一些新增的 tauri API，可以监听文件夹中文件的变动

官方提供了升级工具，可以帮助配置文件相关的变动

### 更新后导致的一些问题

文件相关操作一些 API 需要改动

<br />

### 为什么要升级

因为要新增对文件的操作功能，对原本的文件操作 API 进行修改，如果之后升级，还要再写一遍，避免返工作为考量之一。

其次是新版已经趋于稳定，能避免早期版本导致的一些问题

浏览器兼容性方面没有变化

项目本身就在不断地进行着优化，比如说，使用 tailwind + shadcn 相关组件去替换 antd，以统一样式风格，一般会在书写到相关逻辑时同步进行
