## 应用线路图

应用的未来规划，将要走到哪一步

### stage 1

工具化，作为一个完整的工具，实现工具应有的功能

- [x] 实现本地文件读写
  - [x] 本地文件的重命名，删除等操作
- [x] 面包屑导航，快捷访问路径文件
- [x] 全局配置保存和读取
- [x] 暗色和亮色两个主题
- [x] todo，待办事项模块
  - [x] todo 头部操作栏
  - [x] todo 的底部计数功能
  - [x] todo 单个项目的重要程度选择
  - [x] todo 的四象限切换功能
  - [x] todo 完成项的隐藏功能
- [ ] 实现 markdown 功能
  - [x] markdown 的快速创建表格等复杂内容
  - [ ] markdown 的快捷键
  - [ ] markdown 的大纲和快捷跳转
- [ ] 完整的 i18n 支持
  - [ ] 多语言支持，排名前五的语言（汉语，英语，日本语，葡萄牙语，俄语）
  - [ ] 之后最多再扩展（韩语，德语，法语）这些语言
- [ ] 底部添加信息栏
- [ ] 本地，或者远程大模型对话功能
  - [ ] 检测和安装本地模型，或者远程模型
- [ ] 工作区文件中存储以下内容：icon，文件的树状图
- [ ] 同步 github 上的代码等功能
- [ ] 文件内容搜索功能
- [ ] 工作区的切换
- [ ] 工作区回收站功能
- [ ] 应用问题自动收集 Sentry

### stage 2

简易化，减少用户的操作（鼠标和键盘切换），让应用更加简便，内容更加丰富，应用本身作为一个整体

- [ ] 应用自定义托盘，进行快速操作
- [ ] 如果配置工作区，生成一个文件用于快速打开工作区
- [ ] 键盘操作
  - [ ] a11y
- [ ] 远程备份
- [ ] 应用支持新窗口打开
- [ ] 更多样式风格
  - [ ] 用户自定义样式
- [ ] 实现甘特图
- [ ] 文件可以拖拽排序
- [ ] 可以将外部文件直接拖拽到应用内
- [ ] 保存 history，可以对历史进行指定内容撤回
- [ ] 监听其它文件对于文件的修改，然后更新文件内容
- [ ] 实现流程图，思维导图编辑
- [ ] 跨平台 react-native

### stage 3

智能化，对应用更多的畅想，用户可以自定义更多内容

- [ ] 应用自动进行每周汇总
